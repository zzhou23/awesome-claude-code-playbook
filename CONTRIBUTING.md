# Contributing to Awesome Claude Code Playbook

Thank you for your interest in contributing! This list is a **playbook**, not a catalog — every entry must explain *when* and *why* to use a resource, not just *what* it is.

## Entry Requirements

Every entry **must** include:

1. A working link to the resource
2. A concise description (one sentence)
3. A **"When to use:"** annotation explaining the scenario

### Entry Template

```markdown
- [Name](https://example.com) - One-sentence description. **When to use:** Scenario where this resource shines.
```

### Example

```markdown
- [claude-test-runner](https://github.com/example/claude-test-runner) - Automatically runs relevant tests after code changes. **When to use:** You want Claude Code to validate changes without manually specifying test commands.
```

## What We Accept

- Tools, MCP servers, and integrations that work with Claude Code
- CLAUDE.md templates and configuration patterns
- Workflow patterns with concrete steps
- Tips backed by real-world usage
- Learning resources that teach practical skills

## What We Do NOT Accept

- Self-promotional links without substantial content
- Unmaintained projects (no commits in 12+ months, unless still functional)
- Duplicate entries covering the same tool/pattern already listed
- Entries without a "When to use" annotation
- Paid-only resources without a free tier or trial

## How to Submit

1. **One PR per entry** (or a batch of closely related entries in the same category)
2. Add your entry to the appropriate section in `README.md`
3. Keep entries in alphabetical order within each section
4. Ensure your entry follows the template format above
5. Check that all links are working

## Quality Checklist

Before submitting, verify:

- [ ] Entry follows the `[Name](url) - Description. **When to use:** ...` format
- [ ] Link is valid and points to the correct resource
- [ ] Description is concise (one sentence)
- [ ] "When to use" annotation is specific and actionable
- [ ] Entry is placed in the correct section
- [ ] Entry is in alphabetical order within its section
- [ ] No duplicate of an existing entry

## Updating Existing Entries

If a resource has changed significantly, feel free to submit a PR updating its description or "When to use" annotation. Please explain the change in the PR description.

## Code of Conduct

By contributing, you agree to abide by our [Code of Conduct](code-of-conduct.md).
